# Система учета поголовья скота (CRUD + PostgreSQL Repository)

Профессиональная реализация консольного Python-приложения для управления реляционной базой данных «Животноводство» в рамках лабораторной работы по СУБД. Проект спроектирован с разделением ответственности по слоям (архитектурный паттерн Repository) и строгой валидацией данных.

## Архитектурные особенности проекта

* **Разделение секретов:** Конфигурация приложения вынесена в файл окружения `.env`, защищенный от коммитов в публичный доступ.
* **Слой данных (Repository):** Все SQL-запросы изолированы от бизнес-логики в отдельном классе.
* **Безопасность (SQL-инъекции):** Абсолютно все запросы (INSERT, SELECT, UPDATE, DELETE) являются параметризованными. Конкатенация строк не используется.
* **Строгая типизация (DTO):** Для валидации входных данных и типов используются модели **Pydantic v2**.
* **Автоматическое управление транзакциями:** Реализован кастомный контекстный менеджер для автоматического `commit` при успехе и `rollback` при ошибках СУБД.
* **PEP 8 & PEP 257:** Весь код полностью документирован в стиле Google Style Docstrings и отформатирован по гайдлайнам Python.

## Структура проекта

* `config.py` — Считывание и валидация переменных окружения из `.env` через `pydantic-settings`.
* `database.py` — Создание таблиц (DDL) и менеджер соединений с СУБД.
* `models.py` — Описание Pydantic-моделей данных для совхозов, видов скота и поголовья.
* `repository.py` — Набор CRUD-методов и сложных JOIN-выборок.
* `main.py` — Главный оркестратор и демонстрационный сценарий работы программы.
* `.env.example` — Шаблон конфигурационного файла для развертывания проекта.
* `requirements.txt` — Список внешних зависимостей.
